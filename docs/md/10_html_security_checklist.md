# 第10章: HTMLセキュリティチェックリスト

AI生成HTML、単体配布HTML、サンプルHTMLを開く前と公開前に使う汎用チェックリストです。1つでも説明できない項目があれば、その時点でブラウザ実行を止めます。

## 10.1 使い方

1. まずテキストエディタでHTMLを開きます。
2. このチェックリストを上から順に確認します。
3. 停止条件に当てはまったら、実行前に修正または隔離します。
4. 修正後はネット遮断状態で再確認します。

## 10.2 判定の目安

- PASS: 用途と影響を説明でき、不要な外部通信や不審コードがない
- REVIEW: 用途は分かるが、固定版・ローカル化・CSPなどの改善余地がある
- STOP: 説明できないコード、通信、権限要求、難読化がある

## 10.3 スクリプト確認

- [ ] `<script>` タグの数と配置を把握した
- [ ] 外部JSのURL、ドメイン、用途を説明できる
- [ ] バージョンが完全固定されている
- [ ] インラインJSの内容を説明できる
- [ ] eval / Function / atob が使われていない

停止条件:

- 説明できないJavaScriptが1つでもある
- latest、メジャー版だけ、未固定版の外部JSを使っている

推奨修正:

- 外部JSはローカル配置を優先する
- インラインJSは外部ファイルへ分離する

## 10.4 外部通信の有無

- [ ] fetch / XMLHttpRequest / WebSocket を確認した
- [ ] 外部URLの用途を説明できる
- [ ] 不明なドメインが含まれていない
- [ ] ネット遮断状態で動作確認した

停止条件:

- ローカルHTMLなのに不要な通信がある
- 通信先ドメインの管理者や用途が不明

推奨修正:

- 静的HTMLなら connect-src 'none' をCSPで設定する

## 10.5 外部リソース

- [ ] `script` / `link` / `img` / `iframe` のURLを一覧化した
- [ ] 信頼できる配布元だけを使っている
- [ ] latest や未固定版を使っていない
- [ ] 可能なものはローカル配置した

停止条件:

- 出所不明のCDNや短縮URLを使っている
- バージョン固定なしで配信物に依存している

## 10.6 フォーム・入力

- [ ] `form` の action を確認した
- [ ] 入力値がどこへ送られるか把握している
- [ ] パスワード、トークン、秘密情報の入力欄が必要最小限である

停止条件:

- 送信先が説明できない
- UIでは見えにくい形で値を送信している

## 10.7 自動実行トリガ

- [ ] `onload` / `onerror` などのイベント属性を確認した
- [ ] `setTimeout` / `setInterval` の用途を説明できる
- [ ] ページ表示だけで何が実行されるか説明できる

停止条件:

- 開いた瞬間に不明な処理が実行される

## 10.8 iframe / 埋め込み

- [ ] `iframe` の有無を確認した
- [ ] 埋め込み先URLを把握している
- [ ] 不明な外部サイトを読み込んでいない

停止条件:

- 埋め込み先がブラックボックスで説明できない

## 10.9 難読化・不審コード

- [ ] Base64や不要なエンコードがない
- [ ] 無意味に短い変数名だらけのJSがない
- [ ] 難読化、自己展開コード、読めない圧縮片がない

停止条件:

- 読めないコードがある
- コメントや命名から意図を追えない

## 10.10 ローカル環境アクセス

- [ ] `localhost` や `127.0.0.1` へのアクセスがない
- [ ] 開発中APIや社内向けURLが残っていない

停止条件:

- 開発者PCやローカルサービスを狙う通信がある

## 10.11 権限系API

- [ ] クリップボード、通知APIの利用有無を確認した
- [ ] カメラ、マイク、位置情報などの利用有無を確認した
- [ ] 必要性を説明できる

停止条件:

- 必要性を説明できない権限要求がある

## 10.12 実行環境チェック

- [ ] まずテキストエディタで中身を確認した
- [ ] ネット遮断してブラウザで開いた
- [ ] 必要ならVMやコンテナで検証した

停止条件:

- 中身を見ずにブラウザで開く

## 10.13 安全側の実装修正パターン

- 外部JS/CSSはローカル配備し、バージョンを完全固定する
- `script` をインラインから外部ファイルへ移す
- 不要な通信を `connect-src 'none'` で止める
- `object-src 'none'`、`frame-src 'none'`、`base-uri 'none'` をCSPで制限する
- 変更後にネット遮断環境で再検証する

## 10.14 静的HTML向けCSP例

```html
<meta http-equiv="Content-Security-Policy" content="default-src 'self'; script-src 'self'; connect-src 'none'; img-src 'self' data:; style-src 'self' 'unsafe-inline'; font-src 'self' data:; object-src 'none'; frame-src 'none'; base-uri 'none'; form-action 'self';">
```

補足:

- Mermaid やチャートライブラリのように、SVGやスタイルを動的生成する場合は `style-src` の条件を実運用に合わせて調整する
- HTTPヘッダで設定できる環境なら、meta よりHTTPヘッダを優先する
